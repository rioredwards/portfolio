I built the official website for **Ohm on the Range**, a music, art, and wellness festival held each year at the base of Mount Adams.

---

## Project snapshot

**What it is:** A fast, mobile-first festival site where attendees, volunteers, artists, and vendors can quickly find key info‚Äîand buy tickets.

**What I did:** Led an end-to-end redesign: requirements, brand + design system, implementation, and launch polish.

### Tech stack

- **Next.js** & **React**
- **TypeScript**
- **Tailwind CSS**
- **shadcn/ui** & **Radix UI**
- **particles.js**

---

## Goals and constraints

The site needed to capture the festival's fun, space-cowboy energy‚Äîthink neon desert + cosmic vibes‚Äîwhile staying fast, accessible, and mobile-first.

Most visitors arrive from social posts or word-of-mouth while they're out with friends, not sitting at a laptop. That shaped the UX: clear content hierarchy, thumb-friendly tap targets, skimmable layout, and performance tuned for mobile.

---

## Highlights

### Visual storytelling

The festival is visually stunning, so photography became the centerpiece. I had access to a large media archive and curated a set that tells the story at a glance: daytime scenery, nighttime stage/laser production, and lots of crowd moments (because the social side of a festival is the experience).

The hero fills the viewport with a rotating carousel of full-bleed images. To keep framing consistent, I made separate crops for mobile and desktop on select shots.

Under the hood, the carousel preloads the next image and crossfades between them for a smooth, cinematic feel. I prototyped the transition in a small HTML/CSS sandbox, then translated it into the site.

### Single-source-of-truth for content

One system I'm especially proud of: lineup content is defined once and transformed into three outputs which together boost the site's SEO and accessibility.

The data is stored as typesafe, structured data (JSON+LD). The object contains dates, locations, organizer info, and performer lists all in one place. From that single source, I generate:

- `RenderableLineupItem` (UI): the visual content (titles + images)
- `ScreenReadableLineupItem` (a11y): human-readable strings rendered with sr-only, so screen readers announce meaningful text instead of raw data.
- `StructuredDataForLineupItem` (SEO): JSON-LD embedded via `<script>` tags for rich results.

Because it's fully type-safe, adding the next year's lineup is straightforward: update one file, and the poster image, screen-reader output, and structured data all update together.

Within the first month of launch, the site reached 3,000+ unique visitors which was more than double the previous year's traffic.

Because the site is content-heavy in the places that matter (lineup, FAQs), performance was a feature‚Äînot an afterthought. I targeted strong Lighthouse scores and a snappy feel across routes.

```typescript
// LineupItems data are delegated to their respective components
export function LineupItem({
  render,
  screenReader,
  structuredData,
  year,
}: LineupItemProps) {
  return (
    <>
      {render && <LineupRender lineup={render} year={year} />}
      {screenReader && <LineupForScreenReader lineup={screenReader} />}
      {structuredData && <LineupStructuredData event={structuredData} />}
    </>
  );
}
```

```typescript
// LineupStructuredData component
export function LineupStructuredData({ event }: LineupStructuredDataProps) {
  return (
    <Script
      id={`structured-data-for-${event.name.replaceAll(" ", "-")}`}
      type="application/ld+json"
    >
      {JSON.stringify(event)}
    </Script>
  );
}
```
